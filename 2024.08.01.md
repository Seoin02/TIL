## 아티클

### <객체 지향 프로그래밍의 4가지 특징>

- 출처: https://www.codestates.com/blog/content/객체-지향-프로그래밍-특징
- **객체 지향 프로그래밍**
  - 여러 객체들을 조립해 하나의 프로그램을 개발하는 방법론
  - 결합성을 낮춘 구조로 변경이 쉬운 프로그래밍
  - 장점: 유지보수성 높아짐 / 코드 재사용성 향상
- **추상화**
  - 객체의 공통적인 속성과 기능을 추출해 정의
  - 어떤 객체가 수행해야 하는 핵심 역할을 규정하고, 실제 구현은 각각 객체가 하게 설계하는 것
- **상속**
  - 상위 클래스로부터 확장된 여러 하위 클래스들이 상위 클래스의 속성과 기능을 간편하게 사용하는 것
  - 코드 반복을 최소화하고, 재사용성을 높일 수 있다.
  - 메서드 오버라이딩을 사용해 내용을 재정의 할 수도 있다.
- **다형성**
  - 어떤 객체의 속성이나 기능이 상황에 따라 다른 역할을 수행할 수 있다.
  - 상위 클래스 타입의 참조 변수로 하위 클래스 객체를 참조하는 것 ⇒ Vehicle 타입의 객체 배열을 생성하면 Vehicle 클래스와 상속 관계에 있는 모든 하위 클래스를 담을 수 있다.
  - 하나의 타입으로 여러 타입의 객체를 참조할 수 있다.
- **캡슐화**
  - 서로 연관있는 프로퍼티와 메서드를 하나의 캡슐로 만들어 데이터를 외부로부터 보호하는 것
  - 잘 만들어진 알약은 분해할 필요가 없다.
  - 장점: 정보 은닉
  - 자바스크립트는 캡슐화를 완전하게 지원하지 않는다. 하지만 ES13에서 `private` 필드를 정의할 수 있게 됐다.`private` 필드의 선두에는 `#`을 붙여준다.
  - `private` 필드는 클래스 외부에서 참조할 수 없다. 다만 접근자 프로퍼티를 통해 간접적으로 접근하는 방법은 유효하다.
